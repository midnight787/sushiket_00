# /commit-push — 変更をコミットして push する

このワークフローは、現在の変更を安全にコミットしてリモートへ push します。
**必ず順番どおりに実行し、確認ステップを飛ばさないこと。**

## 手順
1. 「git status」と「git diff --stat」を実行し、何が変わったかを把握する。
2. 変更内容を**日本語で要約**してユーザーに見せる:
   - どのファイルが、どう変わったか（箇条書き）
   - 秘密情報(.env / *.key 等)が含まれていないか確認した結果
3. コミットメッセージ案を**日本語で**提示する（コミット作法に従う）。
4. ユーザーに「この内容でコミット＆pushして良いか」を尋ね、**明示的なYesを待つ**。
   - No の場合は、何を直すか聞いて手順2に戻る。
5. Yes をもらったら、次のコマンドを順に実行:
   - git add -A
   - git commit -m "<合意したメッセージ>"
   - git push origin main
6. push に失敗した場合（リモートが進んでいる等）:
   - **force push は絶対にしない**。
   - 「git pull --no-rebase」でマージを試み、コンフリクトがあれば
     その旨を日本語で報告し、ユーザーの指示を待つ。
   - 解決できたら再度 push する。
7. 完了したら、コミットハッシュと push 先を日本語で報告する。

## 注意
- 上記以外の git コマンド（reset / rebase / force 系）はこのワークフローでは使わない。
- リモート「origin」が未設定なら、push せず「リモート未設定」とだけ報告する。
- push 先のブランチが main 以外なら、その名前に読み替えること。
