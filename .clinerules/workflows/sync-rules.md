# /sync-rules — ルールと設定ファイルの同期ワークフロー

`.clinerules/` 配下の個別ルールファイルが追加または修正された際、エージェント共通ルールである `.agent` および `.AGENTS.md` に変更を反映・同期する。

## 手順

1.  **ルールファイルの編集**:
    *   `.clinerules/` 配下の個別 md ファイル（`00-safety.md` など）を編集または追加する。
2.  **同期スクリプトの実行**:
    *   PowerShell を起動し、プロジェクトルートで同期スクリプトを実行する:
        ```powershell
        .\SyncAgentRules.ps1
        ```
    *   スクリプトが正常終了し、`.agent` および `.AGENTS.md` が上書き生成されたことを確認する。
3.  **生成ファイルの検証**:
    *   `git diff` またはエディタの差分比較機能を使用して、新しく生成された `.agent` および `.AGENTS.md` に想定通りのルール変更が反映されていることを確認する。
4.  **報告とセーブ**:
    *   同期が完了したことをユーザーに日本語で報告する。
    *   変更内容を確定させるため、`/save` または `/commit-push` ワークフローを用いて同期スクリプト、編集したルール、自動生成された `.agent` / `.AGENTS.md` をコミットする。

## 注意事項

*   `.agent` および `.AGENTS.md` は **自動生成ファイル** であるため、直接変更を加えないこと。直接変更しても次回同期時に上書きされて消失する。
*   ルールそのものを追加・修正する場合は、必ず `.clinerules/` フォルダ内の個別ルールファイルを編集すること。
