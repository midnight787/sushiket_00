# 安全規範（常時適用）

## git の破壊的操作
以下は **絶対に自動実行しない**。提案にとどめ、人間の承認を待つ:
- reset --hard / push --force / push -f / clean -fd / branch -D / rebase
- checkout . / restore .（未コミットの変更を消す系）

破壊的操作を提案する時は、日本語で必ず添える:
- これは何をするか
- 元に戻せるか / 戻し方
- 実行して良いか（Yes/No を待つ）

## 全般
- ユーザーが理解していない操作は、先に日本語でかみ砕いて説明する。
- 迷ったら勝手に進めず質問する。
- 秘密情報（APIキー・パスワード・.env）をコミットに含めない。混入を見つけたら警告する。
