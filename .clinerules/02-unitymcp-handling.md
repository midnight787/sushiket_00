# UnityMCP Tool Handling

## ツールの破壊性で扱いを変える

### 可逆・低リスク（提案後すぐ実行してよい / auto-approve許可候補）
- シーン内のオブジェクト新規作成（キューブ等のプリミティブ追加）
- 読み取り系全般（シーン構成の取得、コンポーネント一覧、ログ参照）
- マテリアル・色・Transform の調整（戻せる範囲）

### 破壊的・高リスク（必ず Plan で合意 → Act で目視承認後に実行）
- 既存スクリプトの上書き・全置換・削除
- プレハブの改変・削除
- シーンの削除・大規模な階層変更
- アセット・ファイルの削除
- プロジェクト設定 / Editor 設定の変更
- ビルド実行

破壊的操作の前には、必ず「何を・どのファイルに対して・どう変えるか」を
1〜3行で宣言してから実行する。

## tool groups の運用
- CoplayDev/unity-mcp の tool groups（vfx / animation / ui / testing 等）は、
  **そのタスクで使うグループだけ** 有効化する。
- 全ツールを常時有効にしてコンテキストを膨らませない（DeepSeek-V4-pro は
  1Mコンテキストだが、ツール定義の肥大はコストと誤爆の両方を招く）。

## 接続前提
- UnityMCP サーバーが Connected であることを前提に動く。接続が切れている
  兆候があれば、勝手にリトライを連打せず、状況をユーザーに報告する。
- Unity Editor 側のコンパイルエラー / コンソールログは、破壊的操作の後に
  必ず確認し、エラーがあれば次の操作に進む前に報告する。
